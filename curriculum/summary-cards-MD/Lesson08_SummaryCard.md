## Today's Format

> 1v1 round-robin. Every student competes individually.
> Your auto runs itself. You drive the rest.

---

## Match Structure

| Period | Length | What happens |
|---|---|---|
| **Autonomous** | 30 sec | Your code runs. Hands off the controller. |
| **Transition** | ~30 sec | Pick up controller. Get ready. |
| **Teleop** | 90 sec | You drive. Complete laps. Carry the amplifier. |
| **Endgame** | Last 30 sec | Return amplifier to Earth. Park. |

---

## Scoring Quick Reference

| Action | Points |
|---|---|
| Rubble in Low Zone (auto) | 1 pt each |
| Rubble in High Zone (auto) | 3 pts each |
| Park in Low Zone (auto) | 5 pts |
| Lap completion (teleop) | 5 pts |
| Lap while carrying Amplifier | +3 pts per lap |
| Amplifier returned to Earth (endgame) | 10 pts |
| Low Zone Parking (endgame) | 5 pts |

> **Guaranteed points:** park auto = 5 pts, endgame park = 5 pts.
> **High value:** amplifier return = 10 pts. Prioritize this in endgame.

## Pre-Match Checklist

Before every match:

- [ ] Robot fits in starting position, mechanisms within frame perimeter
- [ ] Elevator at bottom, scoop flat
- [ ] AutoChooser set to your chosen strategy
- [ ] Alliance color marker visible
- [ ] Controller connected and tested

---

## If Something Goes Wrong

| Problem | What to do |
|---|---|
| Auto doesn't run | Check: robot enabled? Auto mode selected? Controller disconnected? |
| Auto overshoots zone | Note the distance. Tell instructor. One constant change allowed between matches. |
| Robot stops responding | Brief interaction allowed — 1 ticket penalty per game manual |
| Teleop drive feels off | Check controller mapping in sim GUI before next match |

---

## The One Rule for Tweaks

**Constants only. Sim first.**

If you change anything between matches:
1. Change only values in `Constants.java`
2. Run it in sim and verify
3. Confirm with instructor before next match
